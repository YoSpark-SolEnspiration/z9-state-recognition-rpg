# Asset Naming Conventions — Z9 Render System

## Required Format

All production render files should be PNG.

Use exact lowercase names for core character files:

```text
donte.png
isaac.png
samantha.png
caleb.png
```

Use exact uppercase DISC form codes for recognition forms:

```text
DD.png DI.png DS.png DC.png
ID.png II.png IS.png IC.png
SD.png SI.png SS.png SC.png
CD.png CI.png CS.png CC.png
```

## Visual Meaning

The file code describes the active state expression, not a separate person.

Example:

```text
CC.png = Caleb expressing the CC recognition form.
```

It does not mean Caleb became a different character.

## Identity Lock

Each final form must preserve:

- same face
- same body proportions
- same character identity
- same core silhouette language

The overlay changes through:

- posture
- energy
- clothing accents
- color family
- symbolic cues

## Family Color Map

```text
D = Red / Fire Sword / Action
I = Purple / Magic / Expression
S = Green / Wood / Support
C = Blue / Hood / Structure
```

## Export Size

Preferred production size:

```text
2048 × 2048 PNG
transparent background
centered full-body subject
```

The app can display smaller PNGs, but final art should be produced at high resolution so it can be reused in reports, cards, and future animation layers.
